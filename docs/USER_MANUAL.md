# NeuronSpectrum GUI

## Назначение программы

NeuronSpectrum GUI — это настольное приложение для Windows, которое объединяет в одном окне основные функции проекта для работы с EEG через LSL:

- поиск доступных LSL-потоков;
- просмотр параметров и metadata потока;
- запись EEG-сессий в `samples.npy` и `timestamps.npy`;
- offline-анализ корреляций Pearson и Spearman;
- live-корреляции по скользящему окну;
- сравнение сохранённых наборов матриц;
- экспорт результатов в `NPY`, `CSV`, `PNG` и `JSON`.

Программа предназначена для работы с EEG-потоками, которые публикуются через Lab Streaming Layer, например из Нейроспектр.NET.

## Состав поставки

После сборки проект содержит:

- исполняемый файл GUI: `dist/NeuronSpectrumGUI/NeuronSpectrumGUI.exe`;
- установщик: `installer_dist/NeuronSpectrumGUI_Setup.exe`;
- исходный код GUI: `main.py` и каталог `neurospectrum_gui/`;
- существующие консольные скрипты в каталоге `scripts/`.

## Системные требования

- Windows 10/11 x64;
- доступ к LSL-потоку EEG для функций записи и live-анализа;
- для offline-анализа достаточно сохранённых файлов `samples.npy` и `timestamps.npy`.

### Вариант 1. Через установщик

5. Запустите программу через ярлык в меню Пуск или через ярлык на рабочем столе.

Можно запускать программу напрямую из папки сборки:

`dist/NeuronSpectrumGUI/NeuronSpectrumGUI.exe`

1. Установите Python и зависимости проекта.

## Главное окно программы

В программе доступны следующие разделы:

- `Потоки LSL`
- `Запись`
- `Проверка записи`
- `Live-корреляции`
- `Сравнение матриц`
- `Журнал`
- `Настройки`
- `О проекте`

## Раздел «Потоки LSL»

Назначение раздела:

- найти доступные LSL-потоки;
- просмотреть список потоков;
- увидеть базовые параметры выбранного потока;
- открыть расширенные metadata в формате JSON и XML;
- сохранить metadata в файлы.

### Порядок работы

1. Укажите таймаут поиска.
2. Нажмите `Найти потоки`.
3. Выберите нужный поток в таблице.
4. Для подробного просмотра нажмите `Показать metadata`.
5. Для сохранения metadata нажмите `Сохранить metadata`.

### Отображаемые параметры

- имя потока;
- `type`;
- `source id`;
- номинальная частота;
- число каналов;
- формат канала;
- `uid`;
- имя хоста.

## Раздел «Запись»

Назначение раздела:

- выбрать LSL-поток;
- начать запись данных EEG;
- остановить запись вручную или по заданной длительности;
- сохранить результаты в файлы.

### Файлы результата

При записи сохраняются:

- `<prefix>_samples.npy`
- `<prefix>_timestamps.npy`
- `<prefix>_meta.json`
- `<prefix>_info.txt`

### Порядок работы

1. Выберите поток.
2. Укажите префикс сохранения.
3. При необходимости задайте максимальную длительность записи.
4. Нажмите `Запустить запись`.
5. Для ручной остановки нажмите `Остановить запись`.
6. Для перехода к результатам используйте `Открыть папку результатов`.

### Что показывается во время записи

- число принятых сэмплов;
- число каналов;
- длительность записи;
- номинальная частота потока.

## Раздел «Проверка записи»

Назначение раздела:

- загрузить записанную EEG-сессию;
- выбрать окно анализа;
- рассчитать матрицы Pearson и Spearman;
- получить краткую статистику;
- сохранить результаты анализа.

### Входные данные

- `samples.npy`
- `timestamps.npy`
- при наличии `meta.json`

### Основные параметры

- частота дискретизации;
- длина окна в секундах;
- начальное смещение окна;
- путь сохранения результатов.

### Выходные данные

В зависимости от настроек сохраняются:

- матрицы Pearson и Spearman в `NPY`;
- матрица `|Pearson - Spearman|`;
- `CSV`-файлы матриц;
- `PNG`-теплокарты;
- `JSON`-отчёт;
- дополнительные диагностические матрицы и таблицы связей.

### Что видно в интерфейсе

- краткая JSON-сводка по анализу;
- тепловая карта выбранной матрицы;
- табличное представление значений.

## Раздел «Live-корреляции»

Назначение раздела:

- подключиться к LSL-потоку в реальном времени;
- накапливать данные в окне;
- периодически пересчитывать корреляции;
- сохранять последний рассчитанный набор матриц.

### Основные параметры

- поток LSL;
- частота дискретизации;
- длина окна;
- шаг обновления;
- таймауты поиска и `pull_sample`;
- путь сохранения.

### Порядок работы

1. Выберите поток.
2. Укажите параметры окна и обновления.
3. Нажмите `Запустить realtime-анализ`.
4. Наблюдайте обновление матриц и статистики.
5. Для завершения нажмите `Остановить realtime-анализ`.
6. Для сохранения последних матриц нажмите `Сохранить последние матрицы`.

### Отображаемые данные

- матрица Pearson;
- матрица Spearman;
- матрица абсолютной разницы;
- краткая статистика по вне-диагональным элементам;
- число полученных сэмплов и число обновлений.

## Раздел «Сравнение матриц»

Назначение раздела:

- сравнить два сохранённых набора результатов;
- оценить различие между offline- и live-результатами или между двумя сессиями;
- сохранить матрицы различий и JSON-отчёт.

### Входные данные

Нужно указать два префикса результатов, для которых существуют файлы:

- `<prefix>_pearson.npy`
- `<prefix>_spearman.npy`

### Результаты

Сохраняются:

- абсолютная разница матриц Pearson;
- абсолютная разница матриц Spearman;
- `CSV` и `PNG` при включённых опциях;
- итоговый `JSON`-отчёт.

## Раздел «Журнал»

В этом разделе выводятся:

- действия пользователя;
- сообщения о запуске задач;
- сообщения об ошибках;
- служебные статусы записи, анализа и сравнения.

Журнал позволяет понять, что именно произошло, даже если операция завершилась с ошибкой.

## Раздел «Настройки»

В настройках можно сохранить значения по умолчанию:

- папка результатов;
- таймаут поиска LSL;
- частота дискретизации;
- длина окна;
- шаг обновления;
- таймаут `pull_sample`.

После сохранения эти параметры автоматически подставляются в разделы программы.

## Форматы данных

### Формат сэмплов

Используется соглашение:

`samples.shape = (n_samples, n_channels)`

Для расчёта корреляций данные приводятся к виду:

`(n_channels, n_samples)`

### Типовые параметры EEG

Для проекта типичен поток:

- 20 каналов;
- 500 Гц;
- окно анализа около 2 секунд.

## Интерпретация результатов

### Pearson

Корреляция Pearson показывает линейную связь между каналами.

### Spearman

Корреляция Spearman показывает монотонную связь и меньше зависит от формы распределения, так как использует ранги.

### Матрица `|Pearson - Spearman|`

Позволяет увидеть, где поведение метрик расходится сильнее всего.

### Диагностические матрицы

В offline-анализе могут дополнительно сохраняться:

- signed diff;
- absolute diff;
- signed ratio;
- absolute ratio;
- mean absolute strength;
- sign agreement;
- absolute dominance;
- таблица пар каналов.

## Типовые сценарии работы

### Сценарий 1. Быстрая запись и проверка

1. Найти поток в разделе `Потоки LSL`.
2. Перейти в `Запись` и записать сессию.
3. Открыть `Проверка записи`.
4. Выбрать `samples.npy` и `timestamps.npy`.
5. Выполнить анализ и сохранить `PNG` и `CSV`.

### Сценарий 2. Наблюдение live-корреляций

1. Найти поток.
2. Открыть раздел `Live-корреляции`.
3. Указать частоту, окно и шаг.
4. Запустить realtime-анализ.
5. Сохранить последний снимок матриц.

### Сценарий 3. Сравнение offline и live

1. Получить offline-результаты.
2. Получить live-результаты.
3. В разделе `Сравнение матриц` указать оба префикса.
4. Выполнить сравнение и сохранить отчёт.

## Возможные ошибки и рекомендации

### LSL-поток не найден

Проверьте:

- запущен ли источник потока;
- совпадает ли сеть;
- достаточно ли времени поиска.

### Во время записи нет сэмплов

Проверьте:

- что выбран правильный поток;
- что источник действительно передаёт данные;
- что поток не завершился до старта записи.

### Ошибка анализа окна

Проверьте:

- что частота дискретизации указана верно;
- что окно не выходит за пределы записи;
- что выбран корректный файл `timestamps.npy`.

### Ошибка сравнения матриц

Проверьте:

- что у обоих наборов результатов есть файлы Pearson и Spearman;
- что размеры матриц совпадают.
